### システム概要の説明（30秒）

```
Review Predictorは、新商品の説明文を入力すると、
10,000人の仮想顧客がどのように評価するかを
シミュレーションするツールです。

バックエンドにはPostgreSQL、そしてベクトル検索のために
pgvector拡張を使用しています。

それでは、実際に動かしてみましょう。
```

---

**【画面: 入力画面を表示】**

```
こちらが入力画面です。

ここに商品の説明文を入力します。
今回は、ハンドメイドアクセサリーの説明文を入力してみます。
```

**【操作: テキストエリアに入力】**

入力例：
```
このアクセサリーは、職人が一つ一つ丁寧に手作りした

天然石を使用し、どんな服装にも合わせやすい
```

### シミュレーション実行（20秒）

**【操作: シミュレーション実行ボタンをクリック】**

```
今、バックエンドで以下の処理が行われています。

1. 入力したテキストを384次元のベクトルに変換
2. データベースに登録された10,000人の顧客と類似度を計算
3. 各顧客の評価を1から5で予測
4. レビューテキストを生成

pgvectorのHNSWインデックスにより、
10,000件の検索が約10ミリ秒で完了します。
```

**【画面: ローディング → 結果画面へ遷移】**

```
約5秒で処理が完了しました。
```

---

### 評価予測アルゴリズムの説明（40秒）

```
ここで、評価がどのように計算されているか説明します。

各顧客には5次元のプロファイルベクトルが設定されています。
- 価格敏感度
- 品質重視度
- デザイン重視度
- ブランドロイヤリティ
- レビュー厳しさ

評価の計算式は以下の通りです。

まず、商品説明と顧客の類似度を0から1の間で計算し、
これを5倍したものを基本スコアとします。

次に、顧客のプロファイルによって調整を行います。
- 品質重視の顧客で類似度が高い場合は+0.8
- ブランド重視の顧客で類似度が高い場合は+0.5
- 価格敏感な顧客は-0.5
- レビューが厳しい顧客は-0.5

最終的に1から5の整数に丸めて評価とします。

このように、単純な類似度だけでなく、
顧客の特性を数式で表現することで、
リアルな評価分布を実現しています。
```

### 技術ポイントの説明（30秒）

```
最後に、このシステムの技術的なポイントを説明します。

データベースはPostgreSQL 15に、
pgvector拡張を追加しています。

これにより、通常のRDB機能に加えて、
ベクトル類似度検索が可能になっています。

HNSWインデックスにより、
計算量はO(log n)で、
10,000件の検索が約10ミリ秒で完了します。

また、トランザクションにより
ACID特性を保証し、
外部キー制約でデータ整合性を維持しています。
```

## 想定Q&A

### Q1: なぜpgvectorを選んだのですか？
```
PostgreSQLの拡張として使えるため、
通常のRDB機能とベクトル検索を
1つのデータベースで実現できるからです。
別途ベクトルDBを用意する必要がありません。
```

### Q2: 10,000人の顧客データはどこから？
```
シードデータとして自動生成しています。
各顧客には384次元の嗜好ベクトルと、
5次元のプロファイルベクトルが設定されています。
```

### Q3: レビューテキストはどう生成していますか？
```
セグメント特性、評価、類似度に基づいて
テンプレートから生成しています。
将来的にはLLMによる生成も検討しています。
```
